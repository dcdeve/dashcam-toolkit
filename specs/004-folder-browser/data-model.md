## New Core Interface: `ScannedFolder`

```typescript
// src/interfaces/folders.ts

export type FolderHealthStatus = 'ok' | 'stale' | 'unknown' | 'checking';

export interface ScannedFolder {
  id: string;
  path: string;
  displayName: string;          // basename(path)
  lastScanAt: Date;
  clipCountAtScan: number;
  filesHashAtScan: string;
  createdAt: Date;
}

export interface FolderHealthResult {
  folderId: string;
  status: Exclude<FolderHealthStatus, 'checking'>;   // resolved status
}

export interface FoldersModule {
  /** List all scanned folders ordered by lastScanAt DESC */
  list(): ScannedFolder[];

  /** Upsert the folder record at the end of a successful scan.
   *  files = video files discovered during the scan (name + size already known). */
  upsertAfterScan(dir: string, files: Array<{ name: string; size: number }>): ScannedFolder;

  /** Run the health check for a single folder (reads filesystem metadata). */
  healthCheck(folderId: string): Promise<FolderHealthResult>;

  /** Remove a folder record and all its associated trips + clips from the library.
   *  Files on disk are NOT touched. */
  remove(folderId: string): void;
}

export type FoldersError =
  | 'FOLDERS_NOT_FOUND'
  | 'FOLDERS_REMOVE_FAILED'
  | 'FOLDERS_DB_ERROR';
```

## Entity Relationship

```
scanned_folders
  id ─────────────────── (no FK to trips; trip-folder assignment is query-time)
  path

clips
  path  ── "LIKE scanned_folders.path || '%'"  ──→  belongs to scanned_folder
  tripId ─────────────────────────────────────────→ trips.id

trips
  id    ←───── clips.tripId
  (no folderId FK — resolved at query time via first clip path prefix)
```

### Trip Count Derivation

`UIFolder.tripCount` is computed when loading the folder grid:

```sql
SELECT COUNT(DISTINCT c.trip_id)
FROM clips c
WHERE c.path LIKE :folderPath || '%'
  AND c.trip_id IS NOT NULL
  AND c.trip_id NOT IN (
    -- exclude trips already claimed by a longer-prefix folder
    SELECT DISTINCT c2.trip_id
    FROM clips c2
    JOIN scanned_folders sf2 ON c2.path LIKE sf2.path || '%'
    WHERE sf2.path != :folderPath
      AND sf2.path LIKE :folderPath || '%'
  )
```

In practice for the MVP, nested folders are rare. The simpler `WHERE c.path LIKE :folderPath || '%'` query is acceptable for the initial implementation; the disambiguation logic can be added if the edge case becomes frequent.

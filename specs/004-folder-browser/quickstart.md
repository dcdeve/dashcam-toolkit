## Development Workflow

### 1. Write the contract test (RED)

```bash
# Create tests/contracts/folders.test.ts
# Run it — it must FAIL (module does not exist yet)
npx vitest run tests/contracts/folders.test.ts
```

### 2. Add the DB table

1. Add `scannedFolders` to `src/core/modules/db/schema.ts`
2. Create `resources/migrations/0002_scanned_folders.sql`
3. Verify migration runs: `npm run dev` (app starts without error)

### 3. Implement the core module

```bash
# Implement src/core/modules/folders/index.ts
# Run contract tests — they should pass (GREEN)
npx vitest run tests/contracts/folders.test.ts
```

### 4. Wire IPC

1. Add `FOLDERS.*` constants to `src/shared/ipc.ts`
2. Create `src/main/ipc/folders.ts`
3. Register in `src/main/ipc/index.ts`
4. Expose via `src/preload/index.ts`

### 5. Integrate scanner → folders

- Call `folders.upsertAfterScan(dir, filesWithSizes)` in `scanner.scan()` after the `importing` phase.

### 6. Renderer

1. Add `UIFolder`, `folderSortOption`, `openRescanDrawer()` to `store.ts`
2. Create `FolderCard.tsx` and `FolderDetailView.tsx`
3. Update `Library.tsx` to show folder grid by default

### 7. CLI parity

- Add `src/cli/commands/folders.ts` with `list` and `health` subcommands
- Register in `src/cli/index.ts`

### 8. Final checks

```bash
make check          # lint + typecheck + full test suite
# Add benchmark result to PR description (health check for 500+ clips)
```

## Acceptance Test Checklist

- [ ] Library opens to folder grid when ≥ 1 folder scanned
- [ ] Library shows empty state with CTA when 0 folders scanned
- [ ] Each folder card shows: name, trip count, clip count, last scanned, health badge
- [ ] Health badge shows `checking` then resolves to `ok / stale / unknown`
- [ ] Adding a file outside the app → folder shows `stale` badge on next grid load
- [ ] Clicking folder card → only that folder's trips are shown
- [ ] Back button → folder grid restored (< 300 ms, no full reload)
- [ ] Rescan button → ImportDrawer opens at confirm/preview step with path pre-filled
- [ ] After rescan → health badge updates to `ok`
- [ ] Delete (⋯ menu) → confirmation dialog → folder + trips removed, files intact
- [ ] Sort control orders cards correctly for all 4 options
- [ ] CLI `dashcam folders list` lists all scanned folders
- [ ] CLI `dashcam folders health <path>` returns correct status
- [ ] `make test-contracts` passes
